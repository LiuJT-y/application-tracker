# 秋招投递追踪器 · 用户指南

个人求职全链路管理工具:**投递看板**(拖拽式状态流转)+ **转化漏斗**(数据洞察)+ **AI 面试复盘**(8 维度评分),配套一个 Chrome 插件,边浏览岗位边一键存进投递记录。

线上地址(海外可直接访问):**https://application-tracker-sepia.vercel.app/**

> 给 Claude Code 的开发上下文、约定和路线图在 `CLAUDE.md`,本文件是给使用者看的。

---

## 这工具能干嘛

| 页面 | 作用 |
| --- | --- |
| **看板** `/board` | 6 列状态:感兴趣 → 已投递 → 笔试 → 面试中 → Offer / 已挂。拖卡片换状态,自动记时间线 |
| **AI 面试复盘** `/review` | 填结构化表单(岗位 / 轮次 / 问答 / 卡壳 / 自我感受),AI 返回 8 维度评分 + 优势 + 改进计划;会结合你这条投递所用的简历一起分析 |
| **数据洞察** `/insights` | 转化漏斗、渠道面试率、简历版本面试率 |
| **简历管理** `/resumes` | 三个 tab:**总表**(把简历拆成可复用条目)+ **简历版本**(组合版本、设默认、传 PDF、看面试率)+ **PDF 画廊**(横滑预览各版本简历、逐页翻看) |
| **设置** `/settings` | 填自己的 LLM key(存浏览器本地,不上传)+ 复制插件接入 token |
| **Chrome 插件** | 在招聘网站抓 JD、算简历匹配分,一键 POST 到投递记录 |

数据按账号隔离,每个用户只看得到自己的投递。

### 🔑 AI key 是怎么不泄露的

- key 由你在**设置页自填**,只存你这台浏览器的 localStorage,**不上传服务器、不入库**。
- 做复盘时,key 随该次请求以请求头发到服务端,服务端用完即弃——**不写数据库、不打日志**。
- 三种模型(GPT / Qwen / DeepSeek)都是 OpenAI 兼容接口,设置页填对应 Base URL + Model 即可任选其一,**用谁的 key 谁付费**。

## 简历管理 · 总表(素材库)

进入路径:看板顶栏「**简历管理**」按钮,或直接访问 `/resumes`。

这是简历管理的地基——先把简历拆成一个个**可复用的条目**(一段实习、一个项目、一段教育、一组技能…),沉淀成自己的「素材库」。将来(后续版本)再从这些条目里**挑选、组合成不同的简历版本**,针对不同岗位投不同的简历,还能关联到投递记录和 AI 复盘。

### 个人信息(一人一份)

页面**顶部**有一张「**个人信息**」卡片,跟简历条目分开管理——它是你简历的抬头,**全局只有一份**(姓名、邮箱、电话、所在地、GitHub 主页、领英主页)。点「+ 填写 / 编辑」填好保存即可,将来生成简历时直接用作页眉。它不属于任何一条「条目」,也不会出现在下面的总表里。

### 怎么用

- **新增条目**:点右上「**+ 新增条目**」,在弹窗里填:
  - **类型**(个人简介 / 经历 / 项目 / 教育 / 技能 / 其他)、**标题**(项目名 / 公司名,必填)
  - 机构 / 公司 / 学校、地点、起止时间(直接写「2024.09」「至今」这类文本即可)
  - **正文 / 要点**(**一行一条**,量化成果更佳)——保存时会**按行拆成一条条要点**单独存储,为将来「生成简历」铺路;逐条编辑的表单会在后续版本加上
  - **标签**(逗号分隔,如 `React, TypeScript, 团队协作`)、**链接**
- **筛选**:顶部一排类型 pill(全部 / 经历 / 项目 / 教育 / 技能…),点一下只看该类型,角标是数量。
- **编辑**:每行点「编辑」,复用同一个弹窗改完保存。
- **删除**:每行点「删除」,会二次确认,删除不可恢复。

> 条目按你的账号隔离,只有你自己看得到。

### 简历版本(从条目组合)

切到简历管理页的「**简历版本**」tab——把总表里的条目挑出来、排好序,组成针对不同岗位的简历版本。

- **新建版本**:点「+ 新建版本」,弹窗里:
  - 左侧勾选要放进这版的素材条目;右侧是「已选」列表,**拖 ⠿ 图标可调整先后顺序**。
  - 填**版本名**(必填)和**备注**(这版主打什么);可勾「**设为默认版本**」。
- **版本卡片**展示:条目数、关联投递数、**面试率**(该版本关联的投递里「到过面试」的占比)、默认徽章。
- **编辑 / 删除 / 设为默认**:卡片下方按钮。删除版本只是把用到它的投递解绑(投递本身保留),不会删投递。
- **每人只有一个默认版本**:把某版设为默认会自动取消其它版本的默认。

### 上传简历 PDF

有两种入口给简历挂 PDF 原件:

- **快速上传**:「简历版本」tab 右上角(「+ 新建版本」左边)点「**↑ 上传 PDF**」,选个 `.pdf`,会**一步建好一个同名版本并挂上这份 PDF**。
- **给已有版本传**:在某张版本卡片上点「＋ 上传 PDF」,给那一版挂 PDF(单文件 **上限 4MB**)。
- **预览**:上传后点「预览」,在浏览器新标签里直接打开这份 PDF。
- **替换 / 删除**:点「替换」换一份;点「删除」(二次确认)移除。
- 只接受 PDF;选了非 PDF 或超过 4MB 会**当场提示报错**,不会静默失败。

> PDF 字节**直接存在数据库里**(不依赖任何对象存储 / 云盘),所以海外 Neon 部署和大陆本机部署**两种方式都通用、开箱即用**。列表加载时只读文件名和大小,不会把整份 PDF 拉下来,翻得很快。传好的 PDF 可在「PDF 画廊」tab 里横滑预览、逐页翻看(见下)。

### PDF 画廊(横滑预览 + 逐页翻看)

切到「**PDF 画廊**」tab——把你所有**传过 PDF 的简历版本**铺成一排封面,快速浏览、翻看:

- **横滑切版本**:鼠标按住左右拖动(或触摸板 / 移动端滑动),每张卡片是该版本简历的**首页封面** + 版本名 + 「共 N 页」+ 默认徽章。
- **点进去逐页看**:点某张封面进入该版本详情,从上到下逐页查看,顶部显示「第 x / N 页」,点「← 返回画廊」回到横滑列表。
- **下载原件**:详情页右上「下载原 PDF」直接存原文件。
- 没传过任何 PDF 时给暗淡占位提示;遇到损坏 / 无法解析的 PDF 会显示明确的失败占位,不会白屏卡死。

> 画廊用 **PDF.js** 在你浏览器本地把 PDF 渲染成图像(不嵌网页 PDF 插件,不连任何外部服务),封面和内页都是**滑到哪渲染到哪**的懒加载,几十页也不卡;渲染结果会缓存,来回切不重复加载。

### 在看板给投递选用哪版简历

简历和投递的关联**在看板侧设置**,不在简历管理页:

- 新建岗位(看板「+ 添加岗位」)或编辑岗位(卡片上 hover 出现的 ✎ 按钮)时,弹窗里有「**简历版本**」下拉,选中即把这条投递标记为用了那一版。
- 新建投递时,若你设了**默认版本**,下拉会**自动预填默认版本**(可改),大多数时候不用手动选。
- 选了版本的卡片底部会显示用的是哪一版,一目了然。

### AI 面试复盘现在会结合简历

做 AI 面试复盘(`/review`)时,系统会按下面的优先级自动找一份简历喂给模型,让点评更贴合你的真实背景:

1. **这条投递在看板上选的简历版本** → 用它;
2. 没选的话 → 用你的**默认简历版本**;
3. 两者都没有 → **不注入简历,复盘行为和以前完全一样**。

注入简历后,模型会额外判断:你的回答和简历经历是否对得上、真实项目有没有讲透、并结合岗位 JD 点出简历与岗位的差距。简历文本只在这一次请求里用一下,**不入库、不进日志**(和你的 LLM key 一样用完即弃)。

## 方案一:直接用线上版(推荐给海外用户)

不用装任何东西,打开就用。

1. 访问 **https://application-tracker-sepia.vercel.app/**
2. 点「注册」建账号(邮箱 + 密码),登录后自动进看板。
3. 想用 AI 面试复盘 → 进**设置页**填你自己的 LLM key(OpenAI 等任意 OpenAI 兼容接口):
   - `API Key`、`Base URL`(如 `https://api.openai.com/v1/chat/completions`)、`Model`(如 `gpt-4o-mini`)
   - 点「测试连接」确认通,再保存。key 只存在你这台浏览器,不会上传。
4. 想用 Chrome 插件采集岗位 → 设置页里**复制插件 token**,填进插件配置即可。插件仓库：https://github.com/LiuJT-y/resume-jd-matcher.git

> ⚠️ 数据库托管在 Vercel + Neon(美国区)。**中国大陆直连这个 URL 可能很慢或打不开**,服务器和数据库都在境外。大陆用户建议走方案二本地部署。

> 🔐 **部署者注意(自己 fork 部署到 Vercel 时)**:**不要**在 Vercel 环境变量里填 `OPENAI_API_KEY` / `DASHSCOPE_API_KEY` / `DEEPSEEK_API_KEY`。否则没填自己 key 的用户做复盘时会悄悄消耗*你*的 key。让每个人都在设置页填自己的 key,你的 key 就绝不会被别人用到。Vercel 上只需配 `DATABASE_URL` 和 `JWT_SECRET`。

## 方案二:本地部署(推荐给中国大陆用户)

整个项目跑在你自己电脑上,数据库可以用 Neon 免费库,也可以用本地 Postgres。无需翻墙访问应用本身(但 Neon 建库环节和调用境外 LLM 仍需要相应网络条件)。

### 前置

- **Node.js 20+**(`node -v` 确认)
- 一个 **Postgres 数据库**,二选一:
  - 注册 [Neon](https://neon.tech) 免费库(最省事,需要能访问 neon.tech)
  - 或本地装 Postgres,连接串形如 `postgresql://user:pass@localhost:5432/jobtracker`

### 步骤

```bash
# 1. 装依赖
npm install

# 2. 准备环境变量
cp .env.example .env
```

编辑 `.env`,至少填这两项:

```bash
# 你的 Postgres 连接串
DATABASE_URL=postgresql://...

# JWT 签名密钥(必填),生成一个随机串:
#   openssl rand -base64 32
# Windows 没有 openssl 可用 PowerShell:
#   [Convert]::ToBase64String((1..32 | ForEach-Object { Get-Random -Max 256 }))
JWT_SECRET=粘贴上面生成的随机串
```

其余(种子账号、CORS、服务端兜底 LLM key)都可选,留空即可——LLM key 推荐启动后在设置页里填。

```bash
# 3. 建表
npm run db:push      # 按 schema 建表

# (可选) 灌一批示例投递,方便先看看界面长啥样;只想用空库就跳过
npm run db:seed

# 4. 启动
npm run dev
```

打开 **http://localhost:3000**,点「注册」建账号(邮箱 + 密码),登录后进看板。

### AI 面试复盘的 key

大陆用户走境内大模型最方便:进**设置页**填 Qwen(阿里百炼)或 DeepSeek 的 OpenAI 兼容接口即可,例如:

| 服务 | Base URL | 示例 Model |
| --- | --- | --- |
| DeepSeek | `https://api.deepseek.com/chat/completions` | `deepseek-chat` |
| Qwen / 阿里百炼 | `https://dashscope.aliyuncs.com/compatible-mode/v1/chat/completions` | `qwen-plus` |
| OpenAI | `https://api.openai.com/v1/chat/completions` | `gpt-4o-mini` |

填完点「测试连接」,通了再保存。key 只存本浏览器。

> 也可以把 key 写进 `.env` 作为服务端兜底(见 `.env.example` 的模型链配置),但设置页里填更安全、更灵活。
